# 생활비서 자동 발행 규칙

## 목표
매일 하나의 주제를 발행하되, 한 콘텐츠를 **3개 연속 페이지**로 구성한다. 페이지 수만 늘리기 위한 얇은 중간 화면은 만들지 않고 각 페이지마다 서로 다른 실질 정보를 제공한다.

## 한 콘텐츠의 고정 구조
1. 대상·기본개념·먼저 확인할 조건
2. 조회·계산·비용·예외조건·실제 진행 순서
3. 준비서류·신청·예약·후속조치·공식 사이트 이동

URL은 다음 구조를 사용한다.

- `/<category>/<slug>/`
- `/<category>/<slug>/2/`
- `/<category>/<slug>/3/`

첫 번째와 두 번째 페이지에는 다음 내용 버튼만 강조하고, 공식 사이트 이동 버튼은 마지막 페이지에만 표시한다. 화면에는 `1단계`, `2단계`, `3단계`, `1 / 3` 같은 진행표를 노출하지 않는다.

## 콘텐츠 품질 기준
- 원문 전체 `body`: 3,000자 이상
- `series.parts`: 정확히 3개
- 각 페이지 본문: 1,000자 이상
- 각 페이지별 제목·설명·핵심요약 3개 이상
- 각 페이지에 내용과 직접 관련된 대표 이미지 1개와 대체텍스트 포함
- 전체 콘텐츠 FAQ 2개 이상
- 공식 출처와 확인일 포함
- 첫 번째와 두 번째 페이지의 출처명은 표시하되 외부 공식 링크는 마지막 페이지에서 제공
- 세 페이지의 문장을 반복하거나 같은 내용을 표현만 바꾸어 재사용하지 않음
- 소제목, 체크리스트, 예외상황, 실제 확인 순서를 포함해 빈약한 화면이 되지 않도록 구성

## 발행 순서
1. `automation/topic-queue.json`에서 `status: queued` 주제 하나를 선택한다.
2. 기존 글과 제목·검색의도·슬러그가 겹치지 않는지 확인한다.
3. 변동성 높은 내용은 발행 당일 정부·공공기관 공식 페이지를 확인한다.
4. 금액·기간·대상 조건은 공고 원문 또는 공식 출처 2개 이상으로 교차 확인한다.
5. 하나의 주제를 세 개의 연속 페이지로 나눠 작성한다.
6. 공식 문구를 복사하지 않고 실제 확인 순서, 헷갈리는 조건, 필요한 준비사항을 재구성한다.
7. 대표 이미지는 저작권상 사용 가능한 이미지 또는 자체 제작 이미지만 사용한다.
8. `npm run check`와 `npm run build`를 모두 통과한 경우에만 커밋한다.
9. 발행 후 주제 큐와 `published-log.json`을 같은 변경에서 갱신한다.

## 금지
- 보험, 보험금, 보험 비교, 보험 가입 권유 콘텐츠
- 정부기관으로 오인시키는 표현·로고
- 지급·승인·환급 결과 보장
- 광고 클릭 유도 문구
- 지역명·연도만 바꾼 대량 문서
- 검수하지 않은 초안 자동 게시
- 같은 문장을 세 페이지에 반복
- 내용 없이 다음 버튼만 있는 중간 페이지
- 첫 번째와 두 번째 페이지에서 공식 신청·조회 사이트로 바로 빠지는 주요 버튼
- 화면에 페이지 진행 숫자나 단계 안내판을 크게 노출하는 구성

## 광고 배치 기준
광고는 본문 정보와 분리하며 다음 내용 버튼 바로 위·아래에 붙이지 않는다. 광고를 버튼, 공식 링크, 조회 결과처럼 보이게 만들지 않는다.
