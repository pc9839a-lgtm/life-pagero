# 생활비서 자동 발행 규칙

## 목표
매일 한 편을 발행하되 문서 수를 늘리기 위한 얇은 글은 만들지 않는다. 각 페이지는 다른 페이지로 이동하지 않아도 핵심 질문에 독립적으로 답해야 한다.

## 발행 순서
1. `automation/topic-queue.json`에서 `status: queued` 주제 하나를 선택한다.
2. 발행 로그와 기존 글을 검색해 제목·검색의도·슬러그 중복을 막는다.
3. 변동성 높은 주제는 발행 당일 정부·공공기관 공식 페이지를 웹에서 확인한다.
4. 공식 출처 1개 이상을 사용한다. 금액·기간·대상처럼 바뀌는 정보는 공식 출처 2개 이상 또는 공고 원문을 확인한다.
5. 본문 1,500자 이상, 핵심요약 3개 이상, FAQ 2개 이상, 출처와 확인일을 포함한다.
6. 공식 문구를 복사하지 않고 실제 확인 순서, 헷갈리기 쉬운 조건, 준비서류를 추가한다.
7. `npm run validate:content`와 `npm run build`를 통과한 경우에만 커밋한다.
8. 발행 후 큐 상태와 `published-log.json`을 같은 커밋에서 갱신한다.

## 금지
- 보험, 보험금, 보험 비교, 보험 가입 권유 콘텐츠
- 정부기관으로 오인시키는 표현·로고
- 지급·승인·환급 결과 보장
- 광고 클릭 유도 문구
- 지역명·연도만 바꾼 대량 문서
- 검토하지 않은 AI 초안 자동 게시
- 단순 이동만 위한 중간 페이지

## 퍼널 원칙
연결 글은 각각 다른 질문을 완결한다. 예: 검사기간 확인 → 검사 준비·예약 → 지연 시 조치. 광고는 이동 버튼 바로 위·아래에 배치하지 않는다.
