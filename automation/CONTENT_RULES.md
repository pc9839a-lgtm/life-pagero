# 생활비서 자동 발행 규칙

## 목표
현재 실제로 신청·사용·접수 중인 자동차 행정과 정부지원 제도를 우선 발행한다. 포털 이용법, 공고문 읽는 법처럼 정책명이 없는 일반론보다 **정책 실명·신청기간·대상·금액·현재 상태가 분명한 콘텐츠**를 먼저 선택한다.

하나의 주제는 3개 연속 페이지로 구성한다. 페이지 수만 늘리는 빈 화면은 만들지 않고 각 페이지마다 다른 질문을 해결한다. 화면에는 `1단계`, `2/3`, 진행률 표시를 노출하지 않는다.

## 콘텐츠 선정 우선순위
1. 현재 신청·접수·사용 기간에 들어온 전국 단위 정책
2. 검색 수요가 큰 자동차 세금·검사·이전등록·보조금
3. 마감일 또는 예산 소진 가능성이 있는 제도
4. 다음 분기에 신청이 시작되는 제도
5. 상시 제도와 계산·조회형 생활행정

`정부지원사업 찾는 법`, `지원자격 읽는 법`, `서류 준비 일반론`처럼 제도명이 없는 글은 새로 발행하지 않는다. 기존 글도 실제 정책 콘텐츠로 교체한다.

## 한 콘텐츠의 고정 구조
- 첫 페이지: 누가 대상인지, 현재 신청기간과 핵심 조건
- 두 번째 페이지: 금액·조회·계산·예외·실제 신청 순서
- 세 번째 페이지: 준비서류·완료 확인·후속조치·공식 사이트 이동

URL:
- `/<category>/<slug>/`
- `/<category>/<slug>/2/`
- `/<category>/<slug>/3/`

첫 번째와 두 번째 페이지에는 다음 내용 버튼을 중심으로 둔다. 공식 신청·조회 사이트 버튼은 마지막 페이지에만 제공한다.

## 품질 기준
- `series.parts`: 정확히 3개
- 각 페이지 본문: 1,000자 이상
- 세 페이지 전체 본문: 3,000자 이상
- 각 페이지 제목·설명·핵심요약 3개 이상
- 각 페이지에 내용과 직접 관련된 이미지 1개와 구체적인 대체텍스트
- FAQ 2개 이상
- 공식 출처의 실제 원문 URL과 자료 확인일
- 검색·완성 글의 자료 기준 영역에서 공식 출처를 클릭해 원문을 확인할 수 있어야 함
- 금액·기간·대상은 발행 당일 공식 페이지 또는 공고 원문으로 확인
- 기간이 바뀌지 않는 설명은 법령·공공기관 상시안내를 활용 가능
- 같은 문장과 핵심 내용을 세 페이지에 반복하지 않음
- 글 목록에는 해당 콘텐츠의 대표 이미지를 표시

## 발행 순서
1. `automation/topic-queue.json`에서 현재성·검색수요가 높은 주제를 선택한다.
2. 기존 글과 제목·검색의도·슬러그 중복을 확인한다.
3. 정부 부처·공공기관·지자체 공식 자료를 웹에서 확인한다.
4. 신청기간·금액·대상·예산소진 여부는 최신 공고를 우선한다.
5. 세 페이지를 각각 독립적인 정보량으로 작성한다.
6. 이미지 사용 가능 여부와 대체텍스트를 확인한다.
7. `npm run check`와 `npm run build`를 통과한 경우에만 커밋한다.
8. 큐와 발행 로그를 같은 변경에서 갱신한다.

## 금지
- 보험·보험금·보험 비교·보험 가입 권유 콘텐츠
- 정부기관으로 오인시키는 이름·로고·표현
- 지급·선정·환급 결과 보장
- 광고 클릭 유도
- 내용 없이 이동 버튼만 있는 페이지
- 화면에 1단계·2단계·3단계 또는 1/3 진행률 노출
- 지역명·연도만 바꾼 대량 문서
- 확인하지 않은 금액·기간·대상
- 오래된 공고를 현재 정책처럼 게시
- 정책명이 없는 일반론 중심 콘텐츠

## 광고 배치
광고는 본문과 명확히 분리한다. 다음 내용 버튼과 공식 사이트 버튼 바로 위·아래에는 붙이지 않고, 광고를 버튼·조회결과·공식 링크처럼 보이게 만들지 않는다.
